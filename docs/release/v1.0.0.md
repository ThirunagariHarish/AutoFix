# Release Notes — AutoFix v1.0.0

**Release date:** 2026-04-19
**Test suite:** 173/173 passing

## Highlights

AutoFix is an autonomous multi-project log monitoring and auto-fix system. Point it at one or
more Git repositories, give it VPS SSH access, and it will: clone/pull each repo, inject
structured logging if missing, launch a Claude Code agent per project inside a dedicated tmux
pane, watch production logs over SSH, diagnose errors, apply fixes, run tests, commit and push
— then verify the deployment. A watchdog thread keeps agents alive and a configurable notifier
pushes events to Slack, Discord, or any webhook endpoint. Everything is driven from a single
`projects.yaml` file and a one-line command: `python autofix.py`.

## How to get started

1. Clone this repo
2. `pip install -r requirements.txt`
3. `cp projects.yaml.example projects.yaml` and edit with your repos + VPS details
4. `python autofix.py`

## Known limitations (v1)

- Maximum 10 projects per config
- No Windows support (requires tmux)
- Auto-fix only applies to known error patterns the Claude agent can diagnose
- No auto-rollback (planned for v2)
- SSH host key verification disabled for unattended operation (`StrictHostKeyChecking=no`)

## Rollback

AutoFix does not modify production infrastructure directly — all code changes are committed and
pushed to the project's Git branch by the Claude agent. To roll back any auto-applied fix:

```bash
# Inside the affected project directory
git log --oneline -10          # identify the auto-fix commit(s)
git revert <commit-sha>        # create a revert commit
git push origin <branch>       # push the revert
```

If the watchdog is running and the revert introduces a regression, the agent will automatically
attempt to diagnose and fix it. Stop AutoFix first if you need a clean manual rollback:

```bash
tmux kill-session -t autofix   # stops all agent panes and the watchdog
```
